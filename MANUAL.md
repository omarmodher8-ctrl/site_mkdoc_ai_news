# ドキュメント運用マニュアル

このマニュアルは `docs` フォルダに記事を追加し、サイト全体の整合性を保つための手順をまとめたものです。

## 基本的な追加手順

1. **カテゴリの選択 / 新規作成**  
   `docs` 配下にあるカテゴリ（例: `ai-generated-news-2025-05`、`tech` など）を確認し、必要に応じて新しいディレクトリを作成します。ディレクトリ名はハイフン区切りの英小文字を推奨します。
2. **Markdown ファイルの作成**  
   カテゴリ直下に記事本体となる Markdown ファイル（`.md`）を作成します。ファイル名は英数字と `_` を推奨します。（例: `cleaning_tips.md`）
3. **対応する `.memo` ファイルの作成**  
   `.md` と同じファイル名で拡張子を `.memo` にしたテキストファイルを作ります。（例: `cleaning_tips.memo`）この内容はページの概要文として使用されます。

## Markdown ファイルの書き方

### 1. YAML フロントマターを必ず記述
ページ上部に `---` で囲んだ YAML フロントマターを必ず置き、以下を設定してください。

```yaml
---
date: 2025-06-29       # 公開日（必須。YYYY-MM-DD 形式）
image: /assets/images/default_ogp.png  # OGP 画像（任意）
author: komiyamma       # 著者名（任意）
---
```

- `date` : トップページや構造化データで使用されます。未設定だとトップページ一覧に表示されません。
- `image` : OGP 用の画像パス。省略時は `extra.default_image` が利用されます。
- `author` : 個別に著者名を出したい場合に設定します。

### 2. 見出しと本文
フロントマターの直後に H1 見出し（`# タイトル`）を書き、本文を記述します。見出しはサイトのページタイトルにも利用されます。

```markdown
---
date: 2025-06-29
---

# 片付けのコツ

本文をここに記述します。
```

## `.memo` ファイルの書き方

`.memo` ファイルには記事の概要文（120〜150 文字程度）を書きます。macros や OGP 説明文として使用されるため、内容が分かる要約を記述してください。

```
少しの工夫で暮らしを整える片付け術を紹介します。日々の習慣に加えたいポイントをコンパクトにまとめました。
```

## ページ構成のカスタマイズ

### `.pages` による並び順制御
`mkdocs-awesome-pages-plugin` を使っているため、各ディレクトリに `.pages` を置くことで並び順・タイトル変更などが可能です。

- ディレクトリタイトルを変更: `title: AIニュース 2025年05月`
- 表示順を制御: `arrange` にファイル名を列挙
- 一時的に非公開にする: `hide: true`

```yaml
# docs/ai-generated-news-2025-05/.pages
title: 生成AIニュース 2025年05月
arrange:
  - cleaning_tips.md
  - goal_setting.md
  - study_tips.md
  - time_management.md
```

```yaml
# 非公開にしたい場合の例
arrange:
  - draft_article.md:
      hide: true
```

## トップページへの反映ルール
- `macros/home.py` の `home_sections` マクロが、`date` と `.memo` の内容を元にカテゴリ別の最新記事を生成します。
- `date` がないページは一覧から除外されます。
- `.memo` の先頭 1 行が概要として表示されます。

## ローカル開発のポイント
- PowerShell で `.\.venv\Scripts\Activate.ps1` を実行して仮想環境を有効化します。
- `mkdocs_serve.ps1` を利用すると仮想環境の有効化と `mkdocs serve` の起動をまとめて実行できます。
- `mkdocs_restart.ps1` は既存の `mkdocs serve` プロセスを停止し、指定したポートで再起動します。

以上の手順に従って記事を追加・更新することで、Cloudflare Pages へのデプロイ時にも整合性の取れたサイトを維持できます。
