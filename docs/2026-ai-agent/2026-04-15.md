過去一週間（2026年4月8日頃〜15日）における指定の**AI Agent**（特に**Claude Code**、**Codex CLI**、**Gemini CLI**、**GitHub Copilot Agent Mode**、**Qwen Code**）に関する主な動きを、X（Twitter）上のユーザー発言とウェブ上の最近の情報からまとめます。古い情報は除外し、各トピックで詳細が十分にわかる具体的な内容を中心に抽出しました。複数の視点から分散して引用しています。

### Claude Code (Anthropic)
**Claude Code**が最も活発に議論されており、**Managed Agents**のリリース、ソースコード漏洩の影響、**Agent Teams**機能（複数エージェントが並行・対話しながら作業）、デスクトップ版のUI改善などが話題です。

- Anthropicが新しく「**Claude Managed Agents**」を発表。企業がAIエージェントを構築・展開する際のインフラを大幅に簡素化し、従来の複雑なフローやコード不要で、英語だけでエージェントを構築可能にした。Claude Code自体を活用したワークスペースでのエージェント展開を容易にし、ビジネスでの自動化を加速させる狙い。
- ソースコード漏洩事件が続報。npmパッケージにソースマップが含まれてしまい、TypeScriptのフルソース（数十万行）が露出。Anthropicのプロプライエタリなエージェント制御手法（ツール呼び出し、メモリ管理、sub-agentオーケストレーションなど）が明らかになり、業界全体に「1年分のヘッドスタート」を与えた形に。一方で顧客データ漏洩はなく、人為的ミスと説明された。
- 新機能「**Agent Teams**」で、Claude Codeがタスクを受け取ると自動的に3つ以上のサブエージェントを生成し、バックエンド・フロントエンド・コードレビュー担当などに分担。ターミナルを分割して並行作業し、互いにメッセージを交換しながら調整。リファクタリングやデバッグで「3ペインに分かれて会話しながら完了」した事例が複数報告され、単独作業の限界を超えると評価。
- デスクトップ版の大幅リデザイン。複数のClaudeセッションを1ウィンドウ内でサイドバイサイド表示可能になり、サイドバーで管理しやすくなった。CLIセッションを`/desktop`コマンドでデスクトップにインポートするワークフローも共有された。

X上では企業ARRの急成長（Claude Code中心に9B→30B規模）や、出力精度向上のための裏技（CLAUDE.mdに「Codexがレビューする」と記述）なども話題になりましたが、詳細な体験談を中心にピックアップ。

### Codex CLI (OpenAI)
**Codex CLI**は更新が続き、リアルタイム機能強化やエコシステムの拡充が目立ちます。Claude Codeとのクロスレビューも実践的に使われています。

- 4月上旬のアップデート（v0.120.0など）で、**Realtime V2**が強化。バックグラウンドでエージェントの進捗をストリーミングし、フォローアップ応答をキューイング。WebRTC経路のデフォルト化やTUIメディアサポートも追加され、CLIでの長時間タスクが安定。
- コミュニティで150以上のツールがマッピングされ、**agent-peer-review**のようなクロスエージェントツール（CodexがClaude Codeの出力をレビュー、逆も）が登場。Claude Code vs Codex vs Gemini CLIの18機能比較表も共有され、言語サポートやサンドボックス、コンテキストウィンドウで差別化が議論。
- 実際の使用感として、Zedエディタ内でのCodex CLIがレートリミット消費が激しい一方、ネイティブCodexアプリ/統合版は安定するという声。複数エージェントの並行ワークフローでの活用も増えている。

### Gemini CLI (Google)
**Gemini CLI**はオープンソースのターミナルエージェントとして頻繁に更新され、無料・高コンテキストが強み。Plan Modeやサンドボックス強化が続いています。

- 最近のリリース（v0.38.0など）で**Dynamic Sandbox Expansion**やMulti-Registry Architectureが導入。Plan Modeがデフォルト化され、キーボードショートカットのカスタマイズも可能に。Google SREが実世界の障害対応で活用している事例も紹介。
- ユーザー体験として、Claude Codeのレビューが甘かったコードをGemini CLIが「コードの残骸が残ってビルドできない」と厳しく指摘した事例が共有され、「Gemini CLIやべえ」との反応。タスク中に突然ハングする問題も指摘されたが、無料でGemini 3.1 Pro相当を使える点が高評価。
- WARPターミナルがClaude Code/Codex/Gemini CLIを第一級サポートし、AIコーディングの「コマンドセンター」化が進む動きも。

### GitHub Copilot (Agent Mode)
**Agent Mode**はターミナル統合やサブエージェント並行実行が強化され、ワークフローに深く組み込まれています。

- 「**Rubber Duck**」新機能が追加。異なるモデルファミリーの第2エージェントがコードをレビューし、「AI slop（低品質コード）」を出荷前に検知。Copilot CLIの自己修正能力を高める。
- Agent Modeでサブエージェントを並行実行可能に。検索ツールをサブエージェント化して複数ツールを反復実行する機能も。Jiraとの連携でCopilot Agentがタスクをキックスタートし、人間がレビュー・調整するワークフローが実践例として共有。
- スキル作成のベストプラクティス（コンテキストの適切化、シンプルな指示、実際のエンジニアリング有用性）に関するブログも登場。

### Qwen Code (Alibaba)
**Qwen Code**はオープンソースのターミナルエージェントとして新しく注目を集め、無料・ローカル実行が可能。Qwen3.6 Plusなどのモデル強化と連動。

- Qwen Codeが正式リリース。Qwenシリーズモデルに最適化されたオープンソースターミナルエージェントで、大規模コードベース理解、自動化作業、Claude Codeライクなエージェントワークフローを提供。Skills/SubAgentsなどのツールが豊富で、無料で毎日1000プロンプト以上利用可能という声も。
- Qwen3.6 Plusのリリースと連動し、1Mトークンコンテキスト、ネイティブマルチモーダル、agentic codingが大幅強化。フロントエンド開発からリポジトリレベル問題解決まで対応し、$3/月の低価格プランでの実用性もテストされている。
- VS Code拡張として「red team（レビュー専用）」で使われ、Claude Codeの代替や補完として比較される事例が増加。

全体として、**Claude Code**のエージェント機能進化と漏洩の影響が最大のトピックで、他はCLIツール間の比較・クロス利用、無料/オープンソースオプションの台頭が目立ちます。複数のエージェントを組み合わせる「マルチエージェントワークフロー」が実践的に広がっている点が共通のトレンドです。情報は流動的なので、最新の公式リリースやGitHubを確認することをおすすめします。
