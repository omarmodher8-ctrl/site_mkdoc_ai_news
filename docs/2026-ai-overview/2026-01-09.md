### 2026年1月9日のAI関連ニュースまとめ

本日（2026年1月9日）のAI関連ニュースを、X（Twitter）の投稿やインターネット上の最新情報から集約しました。主に指定された焦点項目を中心に、信頼できるソースに基づいてまとめています。ニュースは本日または直近のものを優先し、古いものは排除しています。同一アカウントからの引用は分散させるよう配慮しました。

#### Google系AI（Gemini、Jules、NotebookLM）
- **GmailへのGemini 3統合強化**: GoogleがGmailにGemini 3を統合し、AI機能が大幅に向上。主な新機能として、メール会話の要約・質問応答（AI Overviews）、文章作成支援（Help Me Write）、文脈に基づく返信候補（Suggested Replies）、重要メール優先表示（AI Inbox）が追加。本日より米国で順次展開開始で、他地域・言語は今後数ヶ月で拡大予定。[@masahirochaen](https://x.com/masahirochaen/status/2009449283700039952) の投稿で詳細が共有されており、大量のメールを扱う管理者層に特に有用と評価されている。
- **Gemini RoboticsとBoston Dynamicsの提携**: CES 2026でGoogle DeepMindとBoston Dynamicsがパートナーシップを発表。GeminiのマルチモーダルAIをAtlasロボットに搭載し、知覚・推論・ツール使用を強化。物理世界でのAI動作（embodied intelligence）を推進する動きで、工場や危険環境での活用が期待される（PYMNTS.com報道）。
- **Gemini APIの検索機能**: Gemini APIでGoogle Searchをパラメータとして活用可能になり、競争が活発化。[@burkov](https://x.com/burkov/status/2009477338573230448) は、これにより長年API化されなかったGoogle検索が利用しやすくなったと指摘。
- **SamsungのGemini搭載拡大**: SamsungがGemini AIを搭載したモバイルデバイスを2026年末までに8億台に倍増させる目標を発表。中価格帯・低価格帯デバイスにも生成AI機能（リアルタイム翻訳、画像編集、プロアクティブアシスタンス）を普及させる方針（Crescendo.ai報道）。

#### Claude/Anthropic
- **Claude Codeの改善と活用Tips**: Claude Agent SDKで、エージェントが各実行時にノートをファイルに書き込み（成功/失敗/欠落コンテキストなど）、複数実行のノートを分析してハーネスを自動改善可能。[@mattshumer_](https://x.com/mattshumer_/status/2009423273721803219) がこのテクニックを共有し、開発効率向上に有効。
- **ClaudeとGeminiの性能比較**: [@burkov](https://x.com/burkov/status/2009490500747997453) がClaudeの修正失敗を指摘し、Geminiを試用したところ「2年前のモデル並み」と評価。Claude Code 4.5は書籍出版のダイナミクスを変えるほど強力で、ライセンスを無視したリライトが可能と追加コメント。
- **Claude CodeとCursorの統合**: Claude CodeのSkillsがCursorで認識され、エージェント判断による動的適用が可能。[@kinopee_ai](https://x.com/kinopee_ai/status/2009484912810971151) がスクリーンショットを共有し、開発ツールとしての進化を強調。

#### Microsoft系AI（GitHub Copilot、Microsoft Copilot）
- **Office Agentのパワポ生成機能**: MicrosoftのOffice AgentがPowerPoint資料をAIで自動生成可能に。Web検索や画像収集を自律的に行い、デザイン性が高く編集可能なスライドを作成。[@excel_niisan](https://x.com/excel_niisan/status/2009465527903899994) が動画で実演し、ManusやGeminiの類似機能を超える完成度と評価。早期アクセスで順次ロールアウト中。
- **Copilotの拡張**: CES 2026でAMDがRyzen AI 400シリーズを発表し、Microsoft Copilotとの連携を強化。エンタープライズ向けのAI処理を効率化（Crescendo.ai報道）。

#### OpenAI/ChatGPT
- **AIと著作権問題**: Variety誌がJoseph Gordon-Levittのコメントを報じ、大規模言語モデル（LLM）のほとんどが「大量の盗用」で構築されていると指摘。企業は過去の盗用を許すべきでないと主張。[@Variety](https://x.com/Variety/status/2009493838075998716) の動画で議論が拡散中。
- **生成AIの適切な活用**: 企業が生成AIでやるべきでない問いに挑む問題を[@fladdict](https://x.com/fladdict/status/2009420767336030340) がnoteで指摘。実務での誤用を避けるための考察が注目。

#### XのGrok
- **Grok 4の評価**: AIモデルリストでGrok 4がトップ6に選出され、執筆用途で推奨。[@burkov](https://x.com/burkov/status/2009363928091119780) の関連投稿で、Claude Code 4.5が書籍リライトに強力と比較（LinkedIn投稿経由）。
- **交通シェアの変動**: SimilarwebのデータでGrokのトラフィックシェアが3%超え、DeepSeekに迫る勢い。ChatGPTのシェア低下と対照的（Similarweb投稿）。

#### Perplexity関連
- **トラフィックシェア上昇**: PerplexityのグローバルAIトラフィックシェアが2.1%に上昇。ChatGPTの低下（68%）に対し、Gemini（18.2%）やDeepSeek（3.9%）と共に成長（Similarwebデータ）。

#### MetaのLlama/AI関連
- **Llamaの進化**: MetaのLlamaモデルがAIエコシステムで活用拡大。[@hayakawagomi](https://x.com/hayakawagomi/status/2009453701715644857) が社用ChatGPTの出力ムラを指摘し、Llamaベースの代替を検討中。

#### DeepSeek
- **DeepSeek V3.2の評価**: ベンチマークでClaude等と同等の知性を示し、コード生成・ツール駆動エージェントで強み。[@akira_papa_IT](https://x.com/akira_papa_IT/status/2009440606813344111) がGLM-4.7（DeepSeek関連）とCerebrasの組み合わせで1,000トークン/秒の高速推論を報告。
- **シェア変動**: DeepSeekのトラフィックシェアが3.9%に上昇（Similarweb）。

#### その他有力AIモデル/リサーチ
- **Liquid AIのLFM2.5**: 1.2Bパラメータの小型LLMでGPT-3.5レベルを実現。日本語性能が高く、オンデバイスAI向け。[@akira_papa_IT](https://x.com/akira_papa_IT/status/2009427079532790180) がベンチマークを共有し、ローカル実行の速さを動画で実証。

- **Digital Red Queen論文**: MITとSakana AIの共同研究で、LLMを使ったCore Warでの敵対的プログラム進化。[@hardmaru](https://x.com/hardmaru/status/2009451693847761048) がarXivリンクを共有。
- **AI詩の可能性**: [@emollick](https://x.com/emollick/status/2009473781920842168) がAI生成のラットへの詩を推奨し、創造性の深みを議論。
- **NVIDIAのRubinプラットフォーム**: CES 2026でRubinチップを発表。AIスーパーコンピュータ向けで、2026後半にパートナーから提供開始（NVIDIA Newsroom）。

- **CursorとClaude Codeの連携**: AIエディタとしてCursorがClaude CodeのSkillsを活用。[@kinopee_ai](https://x.com/kinopee_ai/status/2009484912810971151) が実装例を共有。
