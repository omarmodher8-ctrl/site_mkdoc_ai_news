過去一週間（2026年2月1日以降）の情報に基づき、AI関連のコーディングツール（特に**Codex**、**Gemini**、**Claude Code**など、クエリで言及の「CodeCode」はおそらくClaude Codeや類似のタイポ/変種を指すと思われる）における**Skills**（エージェントスキル、事前定義された再利用可能な機能やコマンドセット）に関する主な話題をまとめます。

主にX（Twitter）上の開発者発言と、インターネット上の最新記事・GitHubトレンドから抽出。古い情報は排除し、2026年2月上旬の新鮮な内容に限定。

### Codex（OpenAI）のSkills関連の注目点
Codexの最新版（GPT-5.3-Codex）では、**Skills**がデスクトップアプリ（macOS対応）で本格的に活用可能になり、多エージェント同時実行、Worktrees（並行作業ブランチ管理）、自動レビューなどが強化。Skillsは繰り返しタスクの自動化や事前定義コマンドとして機能し、開発者が「背景エンジニア」的に運用できる点が話題。

- 複数の開発者がCodexアプリで**Worktrees**と**Skills**を組み合わせ、効率的に使うと報告。例えば、[@diegocabezas01](https://x.com/diegocabezas01/status/2020345291577450654) は「CodexアプリでGPT-5.3を使うなら、WorktreesとSkillsの2つが必須」と強調。
- CodexをClaude Codeのサブエージェントとして使い、コミット後の自動ReviewをSkillsで実現するワークフローが人気。[@shmily7](https://x.com/shmily7/status/2020339664276832279) は「ClaudeCodeにCodexを助手としてMCP+Skillsでコマンド化、/Commit後に自動Review。最強の2AIが打工してる感覚」と絶賛。
- Skillsの共通化ニーズが高まっており、[@honda_ai_](https://x.com/honda_ai_) は「CodexとClaude CodeでSkillsをどう共通化してる？ 別定義だと変更漏れが発生」と質問。開発者間で議論活発。

Codexアプリのリリース（2月上旬）でSkillsが自動化・スケジュール実行対応になった点が、ReutersやGIGAZINEなどで報じられ、複数プロジェクト同時管理が可能に。

### Claude Code（Anthropic）のSkills関連
Claude Code（Claude Opus 4.6など）ではSkillsがエージェントの拡張として定着。レビュー、プラン立案、UI改善などに活用。

- [@ao2363](https://x.com/ao2363/status/2020367777891271029) はCodex 5.3の速度を活かし、Claude Codeに**ExitPlanMode**（プランReview）、**PreToolUse**（コミット前Review）、**Skills**（レビュー統合）を実装中と共有。無限ループ防止に注意喚起。
- [@ljhspurs](https://x.com/ljhspurs/status/2020305667903705094) は自己進化エージェント構築で、**specs-interview skills**を使いCodex 5.3 + Opus 4.6にアーキテクチャを投げ、対話で洗練。資深アーキテクトをAIに任せられるレベルと評価。
- Skillsのセキュリティ監査も話題で、[@lkr](https://x.com/lkr/status/2020361074453934357) は怪しいGitHub SkillsをClaudeにフル監査させ、安全性を確認する方法を紹介。

### Gemini関連のSkills
Gemini CLIで**Agent Skills**サポートが追加され、Anthropicの標準を採用。Firebase関連などでライブデモあり。

- Gemini CLIのSkillsでFirebase Rules Writerなどを即時作成可能。YouTubeのFirebase After Hoursで紹介され、プロジェクト共有が容易に。
- UI作成でClaude Codeの**frontend-design** Skillsを使い、Gemini-3-ProのデフォルトUIを改善した例も。[@posi_posi8](https://x.com/posi_posi8/status/2020290135926141345) はGeminiで紫背景UIになったものをSkillsでシンプル化、Codexでリファクタと並列処理。

### GitHub上の今週注目Skills話題
GitHubではSkillsの公開レジストリ/コレクションが急増。

- OpenAIの公式Skillsカタログ（github.com/openai/skills）が更新され、数千のコミュニティSkills対応。
- OpenClaw（関連エージェント）のawesome-openclaw-skillsで5,700超のコミュニティSkills公開（2月7日時点）。Codexアカウント切り替えやトレンド監視スキルなど。
- AntigravityやVibe-codingトピックで700超のbattle-tested Skillsバンドルが共有。Claude Code/Cursor/Codex/Gemini CLI対応。
- 個別例として、[@orfizm42](https://x.com/orfizm42/status/2020372179444170764) が3Dプリンタモデル生成用**vibe-modeling** SkillsをGitHub公開（CadQuery + Claude Code + podman自動化）。

全体として、Skillsは「単なるプロンプト」から「再利用可能なエージェント機能」へ移行し、Codex/Claude Code/Gemini間で共通化・セキュリティ意識が高まっている。開発現場では自動レビュー/並列タスク/カスタムワークフローが実用的価値として共有されている。
