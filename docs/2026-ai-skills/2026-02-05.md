過去一週間（2026年1月29日以降）の情報に基づき、AI関連ツールの「CodeCode」「Gemini」「Codex」で使える「Skills」に関する主な発言や更新をまとめます。主にX（Twitter）上の開発者・AIユーザーからの実用的投稿を中心に、詳細な内容を抽出しています。古い情報は除外し、具体的なSkillsの活用例や特徴がわかるものを優先。

### Codex関連のSkills
OpenAIのCodex（特に新macOSアプリ版）でSkillsが注目されており、個人スキルロードや自動化が強化されています。

- **Skillsのインストール・共有が簡単**：ユーザーが作成したSkillsをnpxで追加可能で、Claude、Cursorなど他のエージェントでも利用できる。例として、Arcium Frontend向けSkillsをインストールすると、ClaudeやCodexアプリで即利用可能になる。
  [@sicmundus.dev](https://x.com/p0k_p0k/status/2019368519587234112)

- **CodexアプリのSkillsサポートがおすすめ**：CodexアプリではSkillsの作成がやりやすいと評価。UIアップグレードに加え、Skills対応とスケジュール自動化が追加され、開発効率が向上。macOS専用ランチャーや並列シェル実行、個人スキルのロード（~/.agents/skills）も特徴。
  [@uslab1994](https://x.com/uslab1994/status/2019381239220957686)

- **Codex 0.95.0リリースでSkills強化**：電撃リリースされたバージョンで、並列シェル実行、個人スキルのロード、macOS専用ランチャーが追加。自分専用のAIエンジニアを育てるのに適している。
  [@Rei0_ai](https://x.com/Rei0_ai/status/2019381473447690345)

- **CodexとClaudeのSkills共有の課題**：Claudeはユーザー・プロジェクトレベルでSkillsを持てるが、Codexはユーザーレベル限定。共通ディレクトリでシンボリックリンクを試みたが、プロジェクトレベルの共有が難しいという指摘。
  [@noricha_vr](https://x.com/noricha_vr/status/2019378664417423601)

- **Codex vs Claude Codeの違い**：CodexはWeb上のコンテキストを頻繁に参照する一方、ClaudeはSkillsに依存する傾向が強い。
  [@matthausk](https://x.com/matthausk/status/2019368586020811024)

### Gemini関連のSkills・機能
Gemini（特にGemini 3 Pro/Flash）では、Skillsというより拡張機能やマルチモーダル能力が議論されていますが、直接「Skills」としてローカルツール統合で言及されるケースあり。

- **Google Skills badgesとGemini活用**：Google Skills badges（JuaraGCPなど）取得でGeminiをコーディング支援に使い、楽しく進められたという体験談。
  [@kriqa](https://x.com/kriqa/status/2019349153193828662)

- **Geminiとの組み合わせ**：Gemini CLIやGemini 3 Proを複数モデルサポートのツール（例: Skywork DesktopやCodeCode風ツール）で使う場合、Skills（ドキュメント生成、Webスクレイピング、画像/動画処理など100+種類）が自動推奨され、生産性が高い。Gemini 3 ProはAutoモードで最適モデル選択。
  [@saidul_dev](https://x.com/saidul_dev/status/2019353703686479893)

### CodeCodeや類似ツール（Skywork Desktopなど）のSkills
「CodeCode」直接の言及は少ないですが、類似のローカルAIデスクトップツール（Skyworkなど）で「Skills」が多用されており、Gemini/Claude/Codex連携で100+の生産グレードSkillsが特徴。

- **100+ Skillsの生産性**：Claude 4.5やGemini 3 Proをバックエンドに、Skillsでドキュメント生成、Webビルド、画像/動画作成、データ抽出（例: 温度チャート画像→JSONテーブル変換）、フォルダ全体の意味理解・要約が可能。ローカル実行でプライバシー重視、プロジェクト全体を把握する「Deep Project Awareness」が強み。
  [@mhdfaran](https://x.com/mhdfaran/status/2019373593424195712)  
  [@itsafiz](https://x.com/itsafiz/status/2019382576771932415)

- **コンテンツ認識ファイル理解**：ローカルファイルを意味的にインデックス化し、自然言語クエリ（例: 「churnに関するスライドを探して」）で検索・要約。PDF、ドキュメント、スプレッドシート、画像などを統一解析。
  [@itsafiz](https://x.com/itsafiz/status/2019382564386140194)

- **Agent Skillsのオープン仕様**：CodexやCursor、Copilotなどから参照可能で、ツール非依存。Claude Codeで記憶領域を持つ独自Agent Skillsの活用例あり。
