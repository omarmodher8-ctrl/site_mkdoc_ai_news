過去一週間（2026年3月22日〜29日頃）で、AIコーディングツール（Claude Code / Codex / Gemini関連、CodeCodeは学習プラットフォームとして一部言及あり）の**Skills**（Agent Skills）に関する主な話題を、X上の発言とウェブ/GitHub情報からまとめました。Skillsとは、Markdownファイル（主にSKILL.md）中心のモジュールで、指示・スクリプト・テンプレートをパッケージ化し、AIエージェントがコンテキストに応じて自動ロード・活用する仕組みです。AnthropicのClaude Codeが先駆けで、OpenAIのCodex、GoogleのGemini CLIなども採用・互換性を高めており、オープンスタンダード化が進んでいます。古い情報は除外し、最近の更新や実践例を中心に抽出しました。

### 1. Agent Skillsの共通規格化とクロスツール互換性の進展
Agent Skillsはフォルダ構造（例: ~/.claude/skills/、~/.codex/skills/、~/.gemini/skills/）で管理され、AIがタスク時に動的に活用します。Google DeepMindの実証では、Gemini API開発でこのスキル導入により性能が**28.2%から96.6%**に向上した事例が注目されています。

- **[@noguryu](https://x.com/noguryu/status/2037366260334487349)**: AIエージェントで「共通規格」が生まれていると指摘。Claude CodeもGeminiも同じ構造（SKILL.mdにルール・知識記述、必要時のみロード、補助スクリプト同梱可能）を採用。RAGやファインチューニングとは異なり、整理されたMarkdownが精度を決める時代になると分析。GoogleとAnthropicが同じ結論に至った点が重要。

- GitHubでは**google-gemini/gemini-skills**リポジトリが1日前に更新。Gemini API向けスキル（gemini-api-dev: 開発ベストプラクティス、vertex-ai-api-dev、gemini-live-api-dev: リアルタイムストリーミングなど）を公開。導入コマンド例: `npx skills add google-gemini/gemini-skills --skill gemini-api-dev --global`。これにより、検索・コード実行・メモリなどのツールをエージェントに即追加可能。

- OpenAIのCodex CLIでもSkillsサポートが実験的に進み、~/.codex/skills/フォルダを認識。Anthropicの仕様を基にした互換性が高まっています。

### 3. Codex / Claude Code / GeminiでのSkills実践例とTips
- Codexアプリでは、個人スキルとして「recent-code-bugfix」（過去1週間の自身コミット由来バグを検知・修正・検証）を定義可能。Automationsと組み合わせ、~/.codex/sessionsをスキャンしてスキル改善も自動化。

- セミナー資料自動作成スキル例: Gemini APIを使い、テーマ・対象者を入力するだけでPowerPoint（画像入り）を生成。テンプレートとスキルを事前設定すれば短いコマンドで完了。

- ゲーム制作例: OpenAI Codex + GPTモデルでFF Tactics風スプライト生成。fal aiスキルで画像モデル切り替え・透明処理・8方向生成を効率化。プロンプトとスキルはGitHubで無料公開。

- スキル管理ツール: caliber-ai-org/ai-setupで、Claude Code/Cursor/Codex向けにskills/MCP/configを一括同期。

- 一般論として、Skillsの質がAIプランの価値を左右（「skill issues」が少ないほど高額プランでも効率的）。Vibe coding vs 専門家での差も指摘され、Skillsは「SOP（標準作業手順）」として機能。

### 4. その他の注目点
- Skillsのセキュリティ懸念: 隠しUnicode文字を使ったプロンプトインジェクションの可能性（Gemini/Claude/Grokなどで解釈される場合）。Anthropicが検知を強化した動きも。
- CodeCode自体は主に初心者向けコーディング学習プラットフォームとして言及され、AIツールのSkillsとは別軸ですが、AI生成コードのワークフローでCodexなどと組み合わせる文脈あり。

全体として、今週は**Skillsのオープンスタンダード化とGitHubリポジトリの充実**が目立ち、クロスツール（Claude Code ↔ Codex ↔ Gemini）での再利用性が強調されています。導入は主にローカルフォルダ配置やnpxコマンドで簡単。詳細は各GitHubリポジトリのREADMEやSKILL.mdを参照すると具体的な使い方がわかります。新しいスキルは日々追加されており、自身のワークフローに合わせてカスタム作成・共有が推奨されます。
